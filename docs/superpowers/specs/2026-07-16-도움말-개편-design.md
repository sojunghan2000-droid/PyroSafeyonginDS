관련 요청: 상단바 "?" 도움말 — 내용 업데이트 + 공통/화면별 재배치 + 유사도 기반 검색.

## 배경 / 문제

현재 도움말([lib/ui.py:688 `_help_dialog`](../../../lib/ui.py))은:
- `_HELP_FAQ`([ui.py:559](../../../lib/ui.py)) **평면 20개 Q&A**를 각각 `st.expander`로 나열 — 카테고리 구분 없음.
- **검색 없음** — 원하는 항목을 찾으려면 전부 스캔.
- 내용이 **v1.5~1.7에 머묾** — 최근 변경(v1.8 진입방식 라디오, v1.9 오동작 위치·장소정정, v1.10 오동작 접수 회차) 누락. 다이얼로그 상단 "자주 묻는 질문 **18개**" 하드코딩도 실제 20개와 불일치.

## 1. 데이터 구조 — 카테고리화

`_HELP_FAQ`(평면 tuple 리스트)를 **카테고리 순서 구조**로 교체:

```python
# (카테고리, [(질문, 답변), ...]) 순서 리스트
_HELP_FAQ_BY_CAT: list[tuple[str, list[tuple[str, str]]]] = [
    ("공통", [ ... ]),
    ("대시보드", [ ... ]),
    ("시설 관리", [ ... ]),
    ("안전점검 관리", [ ... ]),
    ("작업 조치 관리", [ ... ]),
    ("보고서", [ ... ]),
    ("관리자 메뉴", [ ... ]),
]
# 검색용 평면 파생 (카테고리 라벨 포함)
_HELP_FAQ_FLAT: list[tuple[str, str, str]] = [
    (cat, q, a) for cat, items in _HELP_FAQ_BY_CAT for (q, a) in items
]
```

### 기존 20개 재분류 (매핑)

- **공통**: 알림 벨 숫자 / 음성 입력(STT) / 데이터 자동 저장 / QR 스캔 '어느 회차?'
- **시설 관리**: QR 코드 발급 / QR 발급·부착 절차 / 위치(spot) 정의 / 장비 ID·spot ID·위치 차이
- **안전점검 관리**: 점검 시작 / 점검 결과 선택 / 화기작업구간 점검 / 가설컨테이너 사무실 점검 / 세부 checklist / 양호인데 세부 항목 / 불량 사유 카탈로그 / 도면에 없는 위치 등록 / 점검 회차(Round)란
- **작업 조치 관리**: 오동작 등록·조치 / 불량 후속 조치
- **보고서**: PDF 출력
- **관리자 메뉴**: (위치 마스터 검증 관련 — '위치(spot) 정의' 항목이 시설 관리·관리자 양쪽 걸치므로 시설 관리에 두고 관리자 항목은 신규 1건 추가)
- **대시보드**: (QR 스캔 진입은 공통에 포함 — 필요 시 대시보드 카테고리는 신규 항목 없으면 생략 가능)

> 구현 주의: 카테고리에 항목이 0건이면 그 섹션 헤더는 렌더하지 않는다(빈 섹션 방지).

### 신규 항목 추가 (v1.8~1.10)

- **[안전점검 관리] 점검 진입 방식이 바뀌었나요? (v1.8)** — [+ Task 추가]·장소정정의 진입이 탭→**라디오**로 변경. 토글/지도 조작 rerun에도 선택이 유지됨.
- **[안전점검 관리] 장비 위치가 틀렸어요 — 정정하려면? (v1.9)** — 점검 시작 → **장소·구역 정정** → QR 스캔 또는 도면 spot 선택. 정정 전에도 장비의 **현재 위치가 파란 마커**로 표시.
- **[작업 조치 관리] 직접 등록 오동작에 점검 ID·작업 ID가 왜 생기나요? (v1.10)** — 등록 시 **"오동작 접수" 회차+Task 자동 발행**·연결. 회차 목록엔 '오동작 접수' 배지로 노출, 점검 KPI·별지5엔 미포함.
- **[작업 조치 관리] 오동작 발생 위치는 어떻게 기록하나요? (v1.9)** — 오동작 등록 시 **도면에서 위치 선택(선택)**. 통합 목록에서 오동작 행이 지적사항과 동일 열 구조(장소=층/구역, 점검종류=시설 구분).
- **[공통] 용어 — 별지5·6·9가 뭔가요?** — 별지5 지적내역서 / 별지6 통보서 / 별지9 오동작 관리대장.

### stale 답변 갱신 (대표)

- "점검 시작" — 진입 방식 탭 표현을 라디오로 정정.
- "도면에 없는 위치 등록" / "위치(spot) 정의" — 탭 명칭·토글 흐름 현행화.
- 버전 태그(vX.Y)는 유지하되 최신 동작과 어긋난 문구만 정정.

## 2. 경량 유사도 검색 함수

`lib/ui.py`에 순수 파이썬 함수 추가(표준 라이브러리 `difflib`, `re`만 사용):

```python
import difflib, re

def _faq_search(query: str, items: list[tuple[str, str, str]]
                ) -> list[tuple[float, str, str, str]]:
    """query와 각 (cat, q, a)의 유사도 점수를 계산해 임계 이상만 내림차순 반환.
    반환: [(score, cat, q, a), ...]"""
    q = query.strip().lower()
    if not q:
        return []
    q_tokens = set(re.findall(r"[0-9a-z가-힣]+", q))
    scored: list[tuple[float, str, str, str]] = []
    for cat, question, answer in items:
        ql, hay = question.lower(), f"{question} {answer}".lower()
        # (1) 부분 문자열 포함
        sub = 1.0 if q in ql else (0.6 if q in hay else 0.0)
        # (2) 토큰 겹침 비율 (질문+답변 토큰 기준)
        h_tokens = set(re.findall(r"[0-9a-z가-힣]+", hay))
        overlap = (len(q_tokens & h_tokens) / len(q_tokens)) if q_tokens else 0.0
        # (3) difflib 문자 유사도 (질문 대상, 오타·부분어 대응)
        seq = difflib.SequenceMatcher(None, q, ql).ratio()
        score = max(sub, overlap, 0.5 * overlap + 0.5 * seq)
        if score >= 0.3:
            scored.append((score, cat, question, answer))
    scored.sort(key=lambda x: x[0], reverse=True)
    return scored
```

- 임계값 0.3·가중치는 구현 시 몇 개 예시 쿼리(오동작/QR/별지5/위치 정정)로 눈검증 후 미세 조정.
- 한글은 음절 런 단위 토큰 + 문자열 포함 + difflib 문자비율 조합으로 부분어·오타를 실용 수준 커버(임베딩 아님, 18~25개 규모에 충분).

## 3. 렌더 변경 — `_help_dialog`

```python
@st.dialog("도움말 — PyroSafe 사용 가이드", width="large")
def _help_dialog() -> None:
    query = st.text_input("검색", key="help_search",
                          placeholder="예: 오동작, QR, 별지5, 위치 정정")
    if query.strip():
        results = _faq_search(query, _HELP_FAQ_FLAT)
        st.caption(f"'{query}' 검색 결과 {len(results)}건")
        if not results:
            st.info("일치하는 항목이 없습니다. 다른 키워드로 검색해 보세요.")
        for _score, cat, q, a in results:
            with st.expander(f"[{cat}] {q}"):
                st.markdown(a)
    else:
        total = len(_HELP_FAQ_FLAT)
        st.caption(f"자주 묻는 질문 {total}개 · 카테고리별. 상단 검색으로 바로 찾을 수 있습니다.")
        for cat, items in _HELP_FAQ_BY_CAT:
            if not items:
                continue
            st.markdown(f"**{cat}**")
            for q, a in items:
                with st.expander(q):
                    st.markdown(a)
```

- 하드코딩 "18개" → **동적 카운트**(`len(_HELP_FAQ_FLAT)`).
- Streamlit 1.51: 검색 입력의 rerun 시 결과 expander는 접힌 채 재표시 — 검색 UX상 정상(버그 아님). 다이얼로그는 위젯 rerun엔 자동 유지.

## 엣지 케이스 / 비고

- 검색 결과 0건 → 안내 문구.
- 카테고리 0건 → 섹션 생략.
- `_HELP_FAQ`를 참조하는 다른 코드가 있는지 확인 — 있으면 `_HELP_FAQ_FLAT`(또는 별칭)로 대체. (grep: 현재 `_help_dialog`만 참조로 확인됨.)
