# 위치·분류 정보 보강 설계 (v1.9)

작성일: 2026-07-15
관련 요청: ① 신규 위치 점검 탭에 소방시설 구분 선택, ② 오동작 등록에 도면 위치 모듈 + 통합 목록 행 재배치

## 배경 / 문제

두 흐름 모두 위치·분류 정보를 담을 데이터 필드가 없어 다운스트림(보고서·집계·추적)에서 분류가 끊긴다.

- **신규 위치 점검**([lib/inspection_dialog.py:536-579](../../../lib/inspection_dialog.py)): 신규 위치 Task는 `위치 설명(room_name)`만 받고 **무슨 소방시설을 점검하는지** 정보가 없다. `InspectionTask`에 `category` 필드 자체가 없음([lib/data.py:219-234](../../../lib/data.py)). → 별지5/6·KPI 카테고리 집계에서 누락.
- **오동작 등록**([lib/inspection_dialog.py:1466-1516](../../../lib/inspection_dialog.py)): `분류/구분/발생일자/발견자/내용`만 받고 **위치 정보가 전무**. `Malfunction` 모델에 위치 필드 없음([lib/data.py:309-322](../../../lib/data.py)). 통합 목록에서 오동작 행은 `location=카테고리`, `category="—"`로 열 의미가 지적사항 행과 어긋남([pages_app/deficiency_manager.py:80-81](../../../pages_app/deficiency_manager.py)).

## 스코프 결정 (확정)

| 항목 | 결정 |
|---|---|
| 신규 위치 — 선택 항목 | **소방시설 구분(카테고리) — 선택(옵션)**. 비우면 순수 위치, 고르면 같이 출력 |
| 오동작 — 위치 입력 방식 | **도면 spot 선택 모듈** |
| 오동작 — 위치 필수 여부 | **선택(옵션)**. 도면 밖/불명 오동작 대응 (미지정 허용) |
| 별지9 PDF 서식 | **불변** — 위치는 내부 추적/화면 표시용 |

## 기능 A — 신규 위치 점검: 층·좌표 표시 + 소방시설 구분(선택)

### A-1. 데이터 (마이그레이션 A)

`inspection_tasks`에 nullable 컬럼 1개 추가:

```sql
ALTER TABLE inspection_tasks ADD COLUMN IF NOT EXISTS category text;
```

- 장비 기반 Task: 기존대로 장비에서 카테고리 파생 (이 컬럼 미사용, NULL).
- 신규 위치(spot) Task: 이 컬럼에 저장.

`InspectionTask` dataclass([lib/data.py:219](../../../lib/data.py))에 `category: str | None = None` 추가.
- `_row_to_task` 매핑에 `category=r.get("category")` 추가.
- `add_task` insert dict에 `"category": t.category` 추가.

### A-2. UI ([lib/inspection_dialog.py](../../../lib/inspection_dialog.py) 신규 위치 픽업 폼, 536-579)

- 픽업 안내 박스에 **층** 명시: 현재 `좌표 (x, y) · 다음 spot ID …` → `층 {floor} · 좌표 (x, y) · 다음 spot ID …`.
- `위치 설명` **위**에 selectbox 추가:
  ```python
  _cat_key = f"add_tsk_temp_cat_{round_id}"
  cat_options = ["— 미지정 —"] + list(EQ_CATEGORIES)
  temp_cat = st.selectbox("소방시설 구분 (선택)", options=cat_options, key=_cat_key)
  ```
  (`EQ_CATEGORIES`는 이미 모듈에 존재 — 소화기·간이소화장치 등)
- `신규 위치 + Task 추가` 제출 시: 선택값이 `— 미지정 —`이 아니면 `category`로 사용, 아니면 `None`.
- `_add_task_map_picker` 반환 dict에 카테고리 포함: `{"type": "empty_spot", "data": new_spot, "category": <str|None>}`. 기존 빈 spot 선택(임시 아님) 경로도 동일하게 카테고리 선택 UI 노출 여부 결정 → **신규 좌표 픽업 케이스에만** 노출(빈 spot은 이미 room_name 존재하므로 동일 selectbox 제공). 구현 시 두 경로 모두 selectbox 노출로 통일.

### A-3. 저장 ([lib/inspection_dialog.py:761-786](../../../lib/inspection_dialog.py))

`sel_empty_spot` 분기에서 `add_task(InspectionTask(..., category=picked_category))`. 장비 분기는 `category=None`.

### A-4. 같이 출력 (표시 touchpoint)

- 회차 상세 Task 목록 / 작업 조치 관리에서 신규 위치 Task에 카테고리를 표시(배지/텍스트).
- 점검 실행 다이얼로그 진입 시 Task.category가 있으면 소방시설 구분 프리필.
- **주의**: 통합 목록(deficiency_manager)은 Deficiency/Malfunction 기반이라 A의 카테고리는 점검 완료 후 Deficiency로 흘러갈 때 반영. 본 스펙 A의 표시는 "회차 상세 Task 목록"까지를 1차 범위로 하고, 리포트 전파는 별도.

## 기능 B — 오동작 위치 + 통합 목록 행 재배치

### B-1. 데이터 (마이그레이션 B)

`malfunctions`에 nullable 컬럼 3개 추가:

```sql
ALTER TABLE malfunctions ADD COLUMN IF NOT EXISTS floor text;
ALTER TABLE malfunctions ADD COLUMN IF NOT EXISTS zone text;
ALTER TABLE malfunctions ADD COLUMN IF NOT EXISTS spot_id text;
```

`Malfunction` dataclass([lib/data.py:309](../../../lib/data.py))에 추가:
```python
floor: str = ""
zone: str = ""
spot_id: str | None = None
```
- `_row_to_malfunction`([lib/data.py:472](../../../lib/data.py)): `floor=r.get("floor") or ""`, `zone=r.get("zone") or ""`, `spot_id=r.get("spot_id")`.
- `add_malfunction`([lib/data.py:1047](../../../lib/data.py)) insert dict: `"floor": m.floor, "zone": m.zone, "spot_id": m.spot_id`.

### B-2. 위치 선택 도면 픽커 (신규 헬퍼)

오동작 다이얼로그는 회차 컨텍스트가 없으므로, `_add_task_map_picker`(회차 전용)를 그대로 못 쓴다. **범용 위치 픽커** 함수를 신설:

```python
def _location_map_picker(key_prefix: str, highlight_category: str | None = None) -> dict | None:
    """도면에서 위치(spot/장비) 1개를 선택. 반환 {floor, zone, spot_id} 또는 None."""
```

- `floor_widget`의 `control_toggle / legend_html / plotly_config / lock_overlay_css` 재사용.
- 층 selectbox → 해당 층 도면 + 등록 장비 마커 + spot 마커.
- `highlight_category`와 일치하는 장비는 **파란색(#2563EB)**, 나머지는 회색(#94A3B8).
- 마커/ spot 클릭 → `{floor, zone(=장비 zone 또는 spot.room_name), spot_id}` 반환.
- 잠금 토글 기본 ON (신규 위치 점검과 동일 UX). 토글 등 rerun 시 선택 상태는 session_state로 유지.

### B-3. 오동작 다이얼로그 ([lib/inspection_dialog.py:1466](../../../lib/inspection_dialog.py))

- `오동작 내용` 아래에 **위치 (선택)** 영역 추가:
  - 안내: "오동작 발생 위치를 도면에서 선택하세요 (선택). 도면 밖/불명 시 비워둡니다."
  - `_location_map_picker("mal_dlg_loc", highlight_category=category)` 호출.
  - 선택 시 성공 배너 + session_state에 픽업된 `{floor, zone, spot_id}` 저장. `위치 미지정` 유지 가능.
- `등록` 제출 시 `Malfunction(..., floor=?, zone=?, spot_id=?)`로 저장 (미선택이면 빈 문자열/None).

### B-4. 통합 목록 행 재배치 ([pages_app/deficiency_manager.py:74-89](../../../pages_app/deficiency_manager.py))

오동작 UnifiedRow 매핑 변경 — 지적사항과 열 의미 일치:

```python
# 변경 전
location=m.category,   # "감지기"
category="—",
# 변경 후
location=(f"{m.floor} / {m.zone}" if (m.floor or m.zone) else "위치 미지정"),
category=m.category,   # "감지기" (소방시설 구분)
```

결과: 오동작 행이 `장소시설=층/구역`, `점검종류=소방시설 구분`으로 지적사항 행과 동일 구조. 위치 미선택 오동작은 `위치 미지정` 표기.

### B-5. 오동작 관리대장 표시 ([pages_app/malfunction_log.py](../../../pages_app/malfunction_log.py))

- 목록 테이블에 위치(층/구역) 표기 추가는 **선택 범위**. 1차 구현은 통합 목록(B-4)만 필수, 관리대장 컬럼 추가는 여력 시.

### B-6. 별지9 PDF — 불변

- 별지9 법정 서식엔 위치 칸이 없으므로 리포트 생성 로직([pages_app/report_center.py](../../../pages_app/report_center.py) 등) 변경 없음.
