# '오동작 접수' 회차·Task 자동 발행 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** [오동작 등록] 직접 등록 시 "오동작 접수" 유형의 회차+Task를 발행해 오동작에 연결하고, 통합 목록에 점검 ID·작업 ID를 실값으로 표시한다.

**Architecture:** 스키마 변경 없이 기존 rounds/inspection_tasks 테이블을 재사용한다. 공유 상수 `MAL_ROUND_TYPE = "오동작 접수"`를 data.py에 두고, malfunction_dialog 제출에서 회차+Task를 Completed로 발행해 Malfunction.task_id에 연결한다. 회차 목록엔 배지로 구분 노출하고, 점검 KPI와 별지5 드롭다운에서는 제외한다.

**Tech Stack:** Streamlit 1.51, Supabase(PostgREST), Python 3.11.

## Global Constraints

- 이 저장소엔 pytest 스위트가 없다. 검증은 ① `python -c "import ast; ast.parse(...)"` 구문, ② 헤드리스 `python -c "from lib import data; ..."` 라운드트립(라이브 Supabase에 insert→load→assert→**반드시 cleanup delete**), ③ 브라우저 프리뷰(port 8511)로 한다.
- **스키마 마이그레이션 없음** — 기존 컬럼만 사용.
- 공유 상수 값은 정확히 `MAL_ROUND_TYPE = "오동작 접수"` (한글, 매직스트링 금지 — 모든 지점에서 `data.MAL_ROUND_TYPE` 참조).
- 발행하는 회차·Task 상태는 **"Completed"** (문자열 그대로). 오동작은 점검 중 발생분이 아니라 **직접 등록분에만** 회차 발행(직접 등록 = malfunction_dialog 경로).
- `lib/` 코드 변경은 Streamlit 재시작해야 반영됨: `taskkill //F //PID <8511 PID>` 후 `preview_start`.
- push 대상은 `main`. UI 카피는 한국어.
